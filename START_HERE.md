## 🔬 КАК РАБОТАЮТ ИССЛЕДОВАНИЯ

1. **Анализ запроса** — AI понимает что нужно подробное исследование
2. **Web search** — поиск информации (если доступно)
3. **LLM анализ** — Groq генерирует детальный отчет
4. **Структура отчета**:
   - Введение
   - Ключевые моменты
   - Факты и даты
   - Технологические аспекты
   - Выводы

## 🔧 ПРОБЛЕМЫ И РЕШЕНИЯ

### Gemini API ошибка (404)
**Проблема**: Модель gemini-1.5-pro не найдена
**Решение**: Используется Groq как основной LLM, Gemini только для мультимодального ввода

### Нет интернета для поиска
**Решение**: LLM сама генерирует подробные отчеты на основе своего знания, не требует внешнего поиска

### Segmentation fault при тесте
**Решение**: Не влияет на работу бота, возможно при завершении Python

## 📈 РАБОТАЮЩИЕ ФУНКЦИИ

### ✅ LLM Routing
- **Gemini 1.5 Pro** — основной провайдер (для мультимодального ввода)
- **Groq Llama-3.3-70b** — для исследований и обычных запросов
- **Ollama** — fallback без интернета

### ✅ RAG-память
- **Qdrant** — векторная база (опционально)
- **Sentence Transformers** — embeddings (опционально)
- **История** — в памяти агента (30 сообщений)

### ✅ Мультимодальный ввод
- **Голосовые сообщения** — Whisper + Gemini Multimodal
- **Текст** — прямая обработка LLM
